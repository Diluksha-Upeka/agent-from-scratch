SECTION 1: Project Documentation
================================================================================
The Smart Home Assistant project aims to create an AI-powered system that can
control various home devices including lights, thermostats, door locks, and
security cameras. The system uses natural language processing to understand
user commands and execute them through a unified API.

Architecture Overview:
- Frontend: React-based web dashboard
- Backend: Python FastAPI server
- AI Engine: Fine-tuned LLM for intent classification
- Device Layer: MQTT protocol for device communication
- Database: PostgreSQL for user data, Redis for caching

SECTION 3: Technical Specifications
================================================================================
API Endpoints:
POST /api/v1/devices/command - Send command to a device
GET  /api/v1/devices/status  - Get status of all devices
PUT  /api/v1/devices/{id}    - Update device configuration
GET  /api/v1/analytics       - Get usage analytics
POST /api/v1/auth/login      - User authentication
POST /api/v1/auth/refresh    - Refresh JWT token

Device Types Supported:
1. Smart Lights - brightness, color, schedule
2. Thermostat - temperature, mode (heat/cool/auto), schedule
3. Door Locks - lock/unlock, access log, temporary codes
4. Security Cameras - live feed, motion detection, recording
5. Smart Plugs - on/off, energy monitoring, schedules
6. Window Blinds - open/close, percentage, schedule

SECTION 7: Frequently Asked Questions (Internal)
================================================================================
Q: What is the project deadline?
A: The target launch date is September 15, with a two-week buffer until October 1.

Q: Which LLM are we using for intent classification?
A: We're using a fine-tuned Gemini Flash model, with fallback to a smaller model.

Q: How do we handle device offline scenarios?
A: The system queues commands and retries when the device comes back online.
   Users receive a notification about the device status.

Q: What's our uptime SLA target?
A: 99.9% uptime for the core API, 99.5% for real-time features.

Q: How is user data protected?
A: All data encrypted at rest (AES-256) and in transit (TLS 1.3).
   PII is stored in a separate encrypted database partition.
